프로그래머스 lv.1 **K번째 수** 문제 풀이입니다.

### 📌 문제 설명
배열 array의 i번째 숫자부터 j번째 숫자까지 자르고 정렬했을 때, k번째에 있는 수를 구하려 합니다.   
예를 들어 array가 [1, 5, 2, 6, 3, 7, 4], i = 2, j = 5, k = 3이라면
1. array의 2번째부터 5번째까지 자르면 [5, 2, 6, 3]입니다.
2. 1에서 나온 배열을 정렬하면 [2, 3, 5, 6]입니다.
3. 2에서 나온 배열의 3번째 숫자는 5입니다.   
배열 array, [i, j, k]를 원소로 가진 2차원 배열 commands가 매개변수로 주어질 때, commands의 모든 원소에 대해 앞서 설명한 연산을 적용했을 때 나온 결과를 배열에 담아 return 하도록 solution 함수를 작성해주세요.
   
### ⚠️ 제한사항
- array의 길이는 1 이상 100 이하입니다.
- array의 각 원소는 1 이상 100 이하입니다.
- commands의 길이는 1 이상 50 이하입니다.
- commands의 각 원소는 길이가 3입니다.

### 📥 입출력 예

| **array** | **commands** | **return** |
| :--- | :--- | :--- |
| [1, 5, 2, 6, 3, 7, 4] | [[2, 5, 3], [4, 4, 1], [1, 7, 3]] | [5, 6, 3] |

---

### 💻 내가 작성한 코드

`Arrays.copyOfRange()`를 활용하여 지정된 범위만큼의 부분 배열을 생성한 뒤, 정렬하여 k번째 요소를 추출하는 방식으로 작성했습니다.

```java
import java.util.Arrays;

class Solution {
    public int[] solution(int[] array, int[][] commands) {
        int[] answer = new int[commands.length];

        for (int i = 0; i < commands.length; i++) {
            int start = commands[i][0] - 1;
            int end = commands[i][1];
            int k = commands[i][2] - 1;

            int[] temp = Arrays.copyOfRange(array, start, end);

            Arrays.sort(temp);

            answer[i] = temp[k];
        }

        return answer;
    }
}
```

### 🔍 복기 및 시행착오 (ArrayList.subList 사용 시 주의할 점)
문제를 풀면서 처음에는 List 자료구조와 subList() 메서드를 사용해보려고 고민했었습니다. 하지만 이 과정에서 Java 자르고 정렬하는 과정의 주요한 특성을 재확인할 수 있었습니다.

### 🚨 ArrayList.subList() 사용 시 겪을 수 있는 함정
1. 원본 데이터 훼손 문제 (View 반환)
    - list.subList()는 새로운 객체를 독립적으로 만드는 것이 아니라, 원본 리스트의 특정 범위를 바라보는 View를 반환합니다.
    - 따라서 subList.sort()로 정렬을 수행할 경우, 원본 list의 데이터 순서까지 엉키는 문제가 발생합니다.
    - 만약 List로 풀고자 한다면 `new ArrayList<>(list.subList(...)`)`와 같이 깊은 복사를 통해 독립된 객체로 만들어야 합니다.
2. Index Offset 계산
    - 문제는 1번째부터 시작하는 1-based index이고, Java의 배열/리스트는 0-based index입니다.
    - subList(from, to) 나 copyOfRange(from, to)에서 to는 미포함(exclusive)이므로, start 인덱스에만 -1 처리를 해줘야 범위가 정확히 맞습니다.


💡 이번 문제를 통해 배운 점
1. 배열 자르기는 Arrays.copyOfRange()가 최선
    - 기본 타입 배열(int[])을 자를 때는 굳이 Wrapper 클래스 비용을 감수하면서 ArrayList로 변환할 필요 없이, 표준 라이브러리의 Arrays.copyOfRange()를 활용하는 것이 원본 훼손 우려도 없고 훨씬 깔끔하다는 점을 다졌습니다.
2. 자바 자료구조의 View 개념 인지
    - subList()처럼 원본 리스트의 참 참조를 공유하는 뷰 메서드를 다룰 때는 Side Effect(부작용)가 발생하지 않도록 주의해야 함을 익혔습니다.
